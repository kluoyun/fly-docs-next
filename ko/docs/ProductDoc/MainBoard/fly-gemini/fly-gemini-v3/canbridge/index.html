CAN브리지

CAN버스는 실시간 응용 프로그램을 위한 직렬 통신 프로토콜 버스입니다. 이 버스는 신호를 전송하기 위해 트와이스트드 페어를 사용할 수 있으며, 세계에서 가장 널리 사용되는 필드버스 중 하나입니다. CAN 프로토콜은 자동차 내의 다양한 구성 요소 간의 통신에 사용되어 비싸고 무거운 배선 하네스를 대체합니다. 이 프로토콜의 강력한 특성 덕분에 다른 자동화 및 산업 응용 분야에도 활용됩니다. CAN 프로토콜의 특징에는 전체성 있는 직렬 데이터 통신, 실시간 지원, 최대 1Mb/s의 전송 속도, 11비트 주소 지정 및 오류 감지 기능이 포함됩니다. 3D 프린터에서는 CAN 버스를 사용하여 프린트 헤드가 메인보드에 연결되는 배선의 수를 줄입니다. 원래는 십여 개의 선이 필요했지만, CAN을 사용하면 단지 네 개의 선만 사용하면 되어 배선의 수를 크게 줄이고 배선 난이도를 낮춥니다. 본 장에서는 CAN 브리지를 사용하는 방법에 대해 간략히 설명합니다.

도구 보드가 메인보드와 CAN으로 연결됨​

请注意工具板电源线不是接到CAN口，是接到12V-24v电源线上

请注意120Ω跳线要接上

메인보드 펌웨어 업데이트​

  • Gemini 메인보드에 번들링된 펌웨어는 USB CAN 브리지 펌웨어 설정이어야 합니다.

CAN 설정 및 ID 검색​

  • CAN을 사용하려면 상위 기기에서 CAN 설정이 되었는지 확인하세요.
주의사항

메인보드가 상위 기기에 연결된 후 아래 작업을 진행해야 합니다.

경고

상위 기기의 can 네트워크, 다리 보드의 CAN 속도, CAN 도구 보드의 속도가 완전히 일치하는지 확인하세요.

장치 검색 전 주의사항​

  • CAN ID를 검색하기 전에 SSH 연결을 먼저 해야 합니다.
  • 네트워크를 통해 SSH에 로그인하고 있는지 확인하세요. 시리얼 포트로 로그인하지 않았는지 확인하세요.
  • UTOC를 연결하거나 CAN 브릿지 펌웨어가烧好的 메인보드를 연결했는지 확인하고, 상위 기기와 데이터 전송이 가능한 데이터 케이블을 사용했는지 확인하세요.

장치 검색​

  • 이제 정상적으로 상위 기기에 로그인했다면 다음 명령어를 입력하여 장치를 검색할 수 있습니다. 다음 중 하나의 경우가 나타날 것입니다.
    • lsusb 명령어를 입력하면 ls 명령어를 찾을 수 없다고 나온다면 아래 명령어를 입력하여 명령어를 설치할 수 있습니다. 
      sudo apt-get install usbutils
    • lsusb 명령어를 입력하면 아무 반응도 없으면 시스템 문제로 해결할 수 없습니다. 다른 시스템으로 교체하거나 문제 없는 시스템을 사용해야 합니다.
    • 아래 이미지와 같은 정보가 표시됩니다. 참고만 하세요. 1d50:606f가 표시되는지 확인하세요.
  • 1d50:606f는 이번에 사용할 장치입니다. 나머지 메시지는 시스템 문제로 인해 불완전하게 표시되거나 아예 표시되지 않을 수 있으니 신경 쓰지 않으셔도 됩니다.
  • 여러 개의 1d50:606f가 표시되면 하나를 제외하는 것이 좋습니다. 그렇지 않으면 후속 플래싱과 펌웨어 연결에 영향을 미칠 수 있습니다. 예를 들어 FLY MINI PAD라면 보드 내장 UTOC를 사용하고 다른 CAN 브릿지 장비는 사용하지 않는 것이 좋습니다.
  • 없다면 데이터 케이블이 잘 연결되었는지, 펌웨어가 올바르게 브러쉬되었는지 다시 확인하세요.
주의사항

1d50:606f가 있을 때만 CAN ID를 검색할 수 있습니다.

CAN ID 검색​

  • 아래 명령어를 입력하여 ID를 검색합니다.
~/klippy-env/bin/python ~/klipper/scripts/canbus_query.py can0
  • ID가 표시되고 마지막에 Application:이 Klipper라고 나오면 이 ID를 바로 사용할 수 있습니다.
  • ID가 표시되고 마지막에 Application:이 CANBOOT 또는 Katapult라고 나오면 펌웨어를 플래싱한 후 사용할 수 있습니다.

CAN ID를 구성 파일에 입력하기​

  • 프린터 웹 인터페이스를 열고 좌측 메뉴의 구성 옵션에서 fluidd일 경우 {…}; mainsail일 경우 기계, 클릭하여 printer.cfg를 찾을 수 있습니다.

  • 검색된 UUID를 복사하여 printer.cfg 구성 파일의 [mcu] 섹션의 canbus_uuid: 뒤에 붙여넣습니다. 저장하고 재부팅하면 메인보드에 연결됩니다. Klipper가 펌웨어 업데이트 필요, ADC 오류 등 메시지를 표시하면 Klipper가 메인보드에 연결된 것입니다.

  • Application:이 Canboot라고 표시되면 구성 파일에 입력할 수 없습니다. Klipper 펌웨어를 플래싱한 후 다음 단계로 진행해야 합니다.
