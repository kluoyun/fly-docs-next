# ファームウェア書き込み説明
    
    * ファームウェアのコンパイルと書き込みには上位機器が必要です
    * SHT36 V3はRS232およびCANの2つの接続方法を使用して上位機器と接続することができます
    * RS232およびCANは`XT30(2+2)-F`を使用して接続する必要があります
    * 以下は`XT30(2+2)-F`データラインのスタイルです
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意事項
* メインボードのファームウェア書き込みにはKatapultファームウェアからKlipperを焼く必要があります
* メインボードに既にKatapultファームウェアが書き込まれていることが確認できた場合、最後のKatapultファームウェアの書き込みは不要です。これはメインボードのKatapultファームウェア書き込みが正常に行われなかった場合の予防措置です
* メインボードのリセットボタンを2回迅速に押すことで、メインボード上のLEDが点滅しているか確認できます。もし点滅している場合は、メインボードがKatapultファームウェアに入っていることを示します
:::

## どの種類のファームウェアが必要かを判断する方法

    * SHT36 V3は以下の方法で上位機器と接続することができます
    * <Button variant="contained" disableElevation href="./can">CANファームウェア</Button>
        * 好み: `Katapult`ファームウェアをブートした後、マザーボードに触れる必要なくファームウェアを更新でき、バス接続をサポートしています
        * 壊れ: `KlipperScreen`と同時に使用すると、ホーム位置のタイムアウトなどのエラーが発生する可能性があります
        * ヒント: CANを使用する場合は、上位機器がCANに対応していることを確認し、UTOCまたはブリッジファームウェアが刷られたマザーボードとの組み合わせが必要です
    * <Button variant="contained" disableElevation href="./rs232">RS232ファームウェア</Button>
        * 好み: 特殊なソフトウェアが不要で、通信が信頼性があり安定しています
        * 壊れ: ファームウェアを再び書き込むには、手動DFUモードに移行するか、RSTをダブルクリックして`Katapult`に入れなければなりません
        * ヒント: G2TまたはUTORと一緒に使用する必要があります
