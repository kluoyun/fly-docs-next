메인보드 ID 읽기 및 확인

팁

웹 페이지 번역을 열 때 구성 설정을 수정하지 마세요

주의사항

웹 페이지 번역을 열 때 구성 설정을 수정하지 마세요

필수 사항

웹 페이지 번역을 열 때 구성 설정을 수정하지 마세요

  • USB ID 획득
  • CANBus UUID 획득
  • RS232 ID 획득

  • 데이터 전송 기능이 있는 데이터 케이블을 준비하세요. 길이가 특히 길다면 마그네틱 링이 있는 데이터 케이블을 사용하는 것이 좋습니다.

    USB 교토 검색​

    장치 검색 전 주의사항​

    • USB ID를 검색하기 전에 먼저 SSH에 연결
    • 네트워크를 통해 SSH에 로그인해야 하며, 시리얼 로그인은 사용하지 마세요
    • USB 펌웨어가 설치된 메인보드가 제대로 연결되어 있고, 데이터 전송 기능이 있는 데이터 케이블이 연결되어 있는지 확인하세요

    장치 검색​

    • 이제 상위 컴퓨터에 정상적으로 로그인했으므로 lsusb 명령어를 입력하여 장치를 검색할 수 있습니다. 다음과 같은 상황이 발생할 수 있습니다
      • lsusb를 입력했을 때 ls 명령어를 찾을 수 없다고 나오면 아래 명령어를 입력하여 설치할 수 있습니다 
        sudo apt-get install usbutils
      • lsusb를 입력한 후 아무 반응이 없다면 시스템 문제로 인해 해결할 수 없습니다. 시스템을 변경하거나 정상적인 시스템을 사용해야 합니다
      • 아래 이미지와 같은 정보가 나타나면 참고용입니다. ID 뒤의 문자만 확인하면 됩니다
    • 1d50:614e는 이번에 사용할 장치입니다. 나머지 안내문은 무시해도 됩니다. 시스템 문제로 인해 정보가 누락되거나 표시되지 않을 수 있습니다
    • 장치가 없으면 데이터 케이블이 제대로 연결되었는지, 펌웨어가 올바르게 설치되었는지 확인하세요
    주의사항

    1d50:614e가 있을 때만 USB ID를 검색할 수 있습니다

    USB ID 검색​

    ls /dev/serial/by-id/*
    • 아래 명령어를 입력하고 엔터를 누릅니다. 모든 것이 정상적이라면 아래에 파란색 ID가 표시됩니다 (아래는 참고용)
    • 명령어를 입력한 후 ID가 표시되지 않지만 lsusb에는 장치가 있는 경우, 상위 컴퓨터나 상위 컴퓨터 시스템을 변경하세요
    주의사항

    여러 ID가 표시되면 메인보드의 MCU 모델에 따라 해당 ID를 선택하세요

    USB ID를 설정에 입력하기​

    • 프린터 웹 인터페이스를 열고 왼쪽 사이드바의 설정 옵션에서: fluidd는 {…} ; mainsail은 기계로 이동하여 printer.cfg를 찾을 수 있습니다

    • 파란색 ID를 복사하여 printer.cfg 설정 파일의 [mcu] 노드의 serial: 에 입력합니다. 저장 후 재부팅하면 메인보드에 연결됩니다. Klipper가 ADC out of range 오류를 표시하는 것은 정상이며, 히트베드와 열센서를 연결하고 노즐과 히트베드의 열센서 핀과 출력 핀을 설정한 후 저장 후 재부팅하면 됩니다

    위험

    주의: 문서에 나타나는 모든 ID는 예시입니다. 실제 각 메인보드의 ID는 다르므로 실제로 얻은 ID로 수정하여 입력하세요

  • ID 검색 전에 XT30 데이터 케이블을 사용하여 도구 보드를 UTOC 또는 브리지 펌웨어가 설치된 메인보드에 연결하세요. 아래는 참고 회로도입니다.
  • 다이얼 스위치가 올바른 위치로 설정되었는지 주의하세요.
  • 전원이 꺼진 상태에서 멀티미터로 CAN H와 CAN L의 저항 값이 약 60Ω인지 확인하세요. 만약 120Ω라면 120Ω 점퍼를 연결해야 하며, 40Ω 정도라면 점퍼를 제거해야 합니다.

    CAN 구성 및 ID 검색​

    • CAN을 사용하려면 상위 컴퓨터가 CAN을 구성했는지 확인하세요.
    알아두기

    메인보드가 상위 컴퓨터에 연결된 후 아래 작업을 수행하세요.

    경고

    상위 컴퓨터의 CAN 네트워크, 브리지 메인보드의 CAN 속도, CAN 도구 보드의 속도가 완전히 일치하는지 확인하세요.

    장치 검색 전 주의사항​

    • CAN ID를 검색하기 전에 먼저 SSH에 연결
    • 네트워크를 통해 SSH에 로그인했는지 확인하세요. 직렬 포트를 통해 로그인하지 마세요.
    • UTOC에 연결하거나 CAN 브리지 펌웨어를 플래시한 메인보드가 연결되어 있고, 데이터 전송 기능이 있는 데이터 케이블이 상위 컴퓨터에 연결되어 있는지 확인하세요.

    장치 검색​

    • 이제 상위 컴퓨터에 정상적으로 로그인되었으니 lsusb 명령을 입력하여 장치를 검색할 수 있습니다. 다음과 같은 상황이 발생할 수 있습니다.
      • ls 명령을 찾을 수 없다고 나오면 아래 명령어를 입력하여 설치하세요 
        sudo apt-get install usbutils
      • lsusb를 입력해도 아무 반응이 없으면 이는 시스템 문제로 해결할 수 없습니다. 시스템을 교체하거나 정상적인 시스템을 사용하세요.
      • 아래 이미지에 있는 정보가 나타나면 참고용입니다. 1d50:606f가 나타나는지 확인하세요.
    • 1d50:606f는 이번에 사용할 장치입니다. 뒤의 메시지는 무시하세요. 시스템 문제로 인해 완전히 표시되지 않거나 아예 표시되지 않을 수 있습니다.
    • 여러 개의 1d50:606f가 있으면 하나를 제외하는 것이 좋습니다. 그렇지 않으면 후속 펌웨어 업데이트와 연결에 영향을 미칠 수 있습니다. 예를 들어, FLY MINI PAD를 사용하는 경우, 장착된 UTOC를 사용하고 다른 CAN 브리지 장비는 사용하지 마세요.
    • 없으면 데이터 케이블이 잘 연결되어 있는지, 펌웨어가 올바르게 플래시되었는지 확인하세요.
    주의사항

    1d50:606f가 있을 때 CAN ID를 검색할 수 있습니다.

    CAN ID 검색​

    • 아래 명령어를 입력하여 ID를 검색하세요
    ~/klippy-env/bin/python ~/klipper/scripts/canbus_query.py can0
    • ID가 나타나고 마지막에 Application:이 Klipper로 표시되면 이 ID를 바로 사용할 수 있습니다.
    • ID가 나타나고 마지막에 Application:이 CANBOOT 또는 Katapult로 표시되면 펌웨어를 플래시해야 사용할 수 있습니다.

    CAN 문제 모음​

    알아두기
    • CAN ID를 찾을 수 없으면 아래 문서를 참조하세요
    • CAN 문제 모음

    CAN의 ID를 구성에 입력하기​

    • 프린터 웹 인터페이스를 열고, 왼쪽 사이드바의 설정 옵션에서 fluidd는 {}를, mainsail은 기계를 찾아 클릭하면 printer.cfg를 찾을 수 있습니다.

    • 검색된 UUID를 복사하여 printer.cfg의 [mcu] 노드의 canbus_uuid: 에 입력하세요. 저장 후 재시작하면 메인보드에 연결됩니다. Klipper가 펌웨어 업데이트가 필요하다고 알리거나 ADC 오류가 발생하면 Klipper가 메인보드에 연결된 것입니다.

      위험

      주의: 문서에 나타나는 모든 ID는 예시입니다. 실제 각 메인보드의 ID는 다릅니다. 실제로 획득한 ID에 따라 수정하여 입력하세요.

  • ID 검색 전에 XT30 데이터 케이블을 사용하여 도구 보드를 UTOR 또는 G2T에 연결하세요. 아래는 참고 회로도입니다.
  • 다이얼 스위치가 올바른 위치로 설정되었는지 주의하세요.

    RS232 ID 검색​

    장치 검색 전 주의사항​

    • RS232 ID 검색 전에 먼저 SSH에 연결하세요
    • SSH에 네트워크 로그인을 사용해야 하며, 직렬 포트 로그인은 하지 마세요
    • UTOR가 잘 연결되어 있고, 상위 컴퓨터와의 데이터 라인이 데이터 전송 기능을 갖추고 있는지 확인하세요
    • G2T를 사용할 경우 시스템이 UART 연결을 지원하는지 확인하세요

    장치 검색​

    • 이제 상위 컴퓨터에 정상적으로 로그인했으므로 lsusb 명령어를 입력하여 장치를 검색할 수 있습니다. 다음과 같은 상황이 발생할 수 있습니다:
      • lsusb를 입력하면 ls 명령어가 없다고 나오면 아래 명령어를 입력하여 설치하세요 
        sudo apt-get install usbutils
      • lsusb를 입력하고도 반응이 없다면 시스템 문제로 인해 해결할 수 없으며, 시스템을 교체하거나 정상적인 시스템을 사용하세요
      • 아래 이미지와 같은 정보가 나타나면 참고용으로만 알아두시면 됩니다. ID 뒤의 문자만 확인하시면 됩니다
    • 1a86:7523는 이번에 사용할 장치에 해당합니다. 이후의 메시지는 무시해도 되며, 시스템 문제로 인해 정보가 완전히 표시되지 않거나 표시되지 않을 수 있습니다
    • 장치가 없다면 데이터 라인이 제대로 연결되었는지, 펌웨어가 올바르게 설치되었는지 확인하세요
    주의사항

    1a86:7523가 있을 때만 아래 명령어로 RS232 ID를 검색할 수 있습니다

    RS232 ID 검색​

    ls /dev/serial/by-path/*
    • ls /dev/serial/by-path/*를 입력해도 ID가 나오지 않지만 lsusb에서는 장치가 보이면 상위 컴퓨터나 상위 컴퓨터 시스템을 교체하세요
    • UTOR 사용 시 세 개의 ID가 출력되며, 다른 UTOR 시스템을 사용한 적이 있으면 여러 ID가 검색될 수 있습니다
    • ID 뒤에 -port0가 붙어 있어야 사용할 수 있습니다
    주의사항

    도구 보드와 UTOR가 반드시 접지되어 있어야 합니다!!! 그렇지 않으면 도구 보드와 연결할 수 없습니다!!!

    RS232의 ID를 설정에 입력​

    • 프린터 웹 인터페이스를 열고, 왼쪽 사이드바의 설정 옵션에서: fluidd의 경우: {…} ; mainsail의 경우: 기계, 클릭하여 printer.cfg를 찾을 수 있습니다

    • 파란색 ID를 복사하여 printer.cfg 설정 파일의 [mcu] 노드에 serial: 에 입력하세요. 저장 후 재부팅하면 메인보드에 연결됩니다. Klipper가 ADC out of range라고 알려주면 정상적인 현상이며, 히트베드와 열 감지기를 연결하고 노즐 및 히트베드의 열 감지 핀과 출력 핀을 설정한 후 저장하고 재부팅하면 됩니다.

    주의사항
    • RS232 ID는 baud:와 restart_method:command를 설정해야 합니다
    • 아래는 참고용 설정입니다
    [mcu SHT36]
    serial: /dev/serial/by-path/platform-1c1c000.usb-usb-0:1.2:1.0-port0
    baud: 250000
    restart_method:command
    위험

    주의: 문서에 나타난 모든 ID는 예시이며, 실제 각 메인보드의 ID는 다를 수 있습니다. 실제로 얻은 ID에 따라 수정하여 입력하세요.

    위험

    아래 ID는 사용할 수 없습니다!!!!!

    serial: /dev/serial/by-id/usb-1a86_USB_Serial-if00-port0
