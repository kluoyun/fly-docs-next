# ファームウェア焼込み説明
    
    * ファームウェアのコンパイルと焼込みには上位機が必要です
    * SHT36 V3はRS232とCANの2つの接続方法で上位機と接続できます
    * RS232とCANは`XT30(2+2)-F`を使用して接続します
    * 下記は`XT30(2+2)-F`データラインのスタイルです
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意事項
* メインボードのファームウェア焼込みにはKatapultを使用してKlipperを焼込みます
* メインボードにKatapultファームウェアが既に焼込まれている場合は、最後のKatapultファームウェア焼込みステップは不要です。それは主にメインボードのKatapultファームウェア焼込み失敗を防ぐためのものです
* メインボードにKatapultファームウェアが正常に焼込まれている場合、リセットボタンを二回素早く押すと、メインボード上のLEDが点滅していることを確認できます
:::

## どの種類のファームウェアが必要か判断する方法

    * SHT36 V3は以下の方法で上位機と接続できます
    * <Button variant="contained" disableElevation href="./can">CANファームウェア</Button>
        * 利点：`Katapult`ファームウェアを焼込み後、マザーボードに触らずにファームウェアを更新でき、バス接続がサポートされます
        * 欠点：`KlipperScreen`と併用すると、ホーム位置タイムアウトなどのエラーが発生する可能性があります
        * ヒント：CANを使用する場合は、上位機がCANに対応しているか確認し、UTOCまたはブリッジファームウェアを焼込んだマザーボードとの組み合わせが必要です
    * <Button variant="contained" disableElevation href="./rs232">RS232ファームウェア</Button>
        * 利点：特別なソフトウェアなしで使用可能、通信が信頼性高く安定しています
        * 欠点：再びファームウェアを焼込むには、手動DFUモードに入ったり、RSTボタンをダブルクリックして`Katapult`に入れなければなりません
        * ヒント: G2TまたはUTORと組み合わせて使用する必要があります
