펌웨어 설명

주의사항
  • Klipper 펌웨어는 가능하면 당신의 호스트 컴퓨터 펌웨어를 사용하여 컴파일하는 것이 좋습니다
  • 다른 사람이 컴파일한 펌웨어를 사용하거나 펌웨어 버전이 다르면 다양한 문제가 발생할 수 있습니다
  • 가능한 한 커스텀된 호스트 컴퓨터를 사용하지 않는 것이 좋습니다

특별 설명​

특별 설명
  • FLY의 대부분의 메인보드는 출하 시 Katapult 펌웨어가 이미 번인되어 있습니다
  • FLY 메인보드는 현재 Katapult 펌웨어를 통해 Klipper 펌웨어를 업데이트합니다
  • DP5, D5, D7 이 세 메인보드는 해당 Katapult 펌웨어를 수동으로 번인해야 합니다
  • 이전 펌웨어의 영향으로 Katapult 펌웨어를 번인한 후, 메인보드 ID를 검색하려면 메인보드를 두 번 재설정해야 합니다, 그렇지 않으면 펌웨어 번인이 불가능합니다

펌웨어 컴파일 설명​

팁
  • FLY 메인보드는 Katapult와 메모리 카드에 번인된 부트로더를 사용합니다
  • Katapult 펌웨어의 장점은 메인보드에 접촉하지 않고 직접 펌웨어를 업데이트할 수 있다는 점입니다
  • 부트로더 펌웨어의 장점은 잘못된 펌웨어를 업데이트하지 않아도 된다는 점입니다
  • 펌웨어 설정은 MobaXterm을 사용하는 것이 좋습니다, 일부 특별한 SSH 소프트웨어가 컴파일 실패를 초래할 수 있다는 사용자 피드백이 있습니다
  • 어떤 펌웨어든지 컴파일이 정확하게 이루어졌는지 확인해야 합니다, 그렇지 않으면 정상적으로 프린팅이 되지 않거나 장치를 찾을 수 없게 됩니다

펌웨어 컴파일 설명​

팁
  • Enable extra low-level configuration options 이 옵션을 선택해야 구체적인 내용이 표시됩니다

STM32​

  • STM32 펌웨어를 컴파일할 때 GPIO pins to set at micro-controller startup을 설정해야 한다면 반드시 대문자로 작성해야 합니다

RP2040​

  • RP2040 펌웨어를 컴파일할 때 GPIO pins to set at micro-controller startup을 설정해야 한다면 반드시 소문자로 작성해야 합니다

펌웨어 번인 설명​

주의사항
  • Katapult 펌웨어를 통해 Klipper 펌웨어를 번인하려면 장치를 찾을 수 있어야 합니다, 장치를 찾을 수 없다면 메인보드를 수동으로 두 번 재설정해야 합니다
  • 가이드 외의 방법으로 펌웨어를 번인하지 마십시오
  • 펌웨어에 컴파일 오류가 있는지 확인하세요, 컴파일 오류가 있으면 정상적으로 펌웨어를 번인할 수 없습니다

Katapult 펌웨어​

  • 컴파일 오류가 있는 펌웨어를 번인한 후의 해결 방법
  • 메인보드를 두 번 재설정하면 됩니다, 재설정 후 정상적으로 LED가 깜빡여야 합니다
  • 메인보드 펌웨어를 지우고 다시 Katapult 펌웨어를 번인합니다

부트로더 펌웨어​

  • 컴파일 오류가 있는 펌웨어를 번인한 후의 해결 방법
  • 다시 TF 카드를 사용하여 번인하면 됩니다
  • 번인이 되지 않으면 먼저 전원을 끄고 메모리 카드를 메인보드에 설치한 후 전원을 켜면 다섯 초 이내에 번인됩니다
