FLYOS-FAST 시스템 번들

시스템 번들​

FLYOS-FAST 시스템 번들 튜토리얼: flyos-fast 시스템 번들 튜토리얼

















































아이야! 아래 문서는 업데이트 중입니다. 참고하지 마십시오!!​

시스템 소개​

  • FlyOS-Fast는 3D 프린팅에 최적화된 가벼운 Linux 시스템으로, 부팅 속도가 빠르며 켜는 시간만 30초 밖에 걸리지 않습니다. FLY의 모든 시리즈 상위 기계 플랫폼을 지원합니다.
  • 커널 최적화로 Klipper의 실시간 응답 성능을 향상시킵니다.
  • 파일 시스템 최적화로 직접 전원이 꺼져도 시스템이 손상되지 않도록 합니다.
  • Klipper, RRF, Mainsail, Fluidd, Moonraker, KlipperScreen 등 다양한 서비스를 미리 설치합니다.
  • Klipper와 RRF 시스템 전환을 지원합니다.
  • Mainsail과 Fluidd 온라인 한 번의 클릭으로 전환 가능.
  • USB 플래그로 인쇄 가능.
  • KPPM 모듈과 함께 사용하면 전원이 끊겼을 때 다시 인쇄할 수 있습니다.
  • Fluidd 등 프론트엔드 UI의 기본 언어, 기본 프린터 이름 설정 가능.
  • 추가 OEM 맞춤 기능: 전원 로고 설정, 사전 설정된 구성 파일 설정, Mainsail 탭 숨기기 설정 등은 저희에게 문의하세요.

준비 작업​

  1. 시스템 이미지 다운로드

    • 이미지 링크: Piv2 시스템 이미지 다운로드 (Windows 전용)
    • 이미지 링크: Piv2 시스템 이미지 다운로드 (Windows, MacOS, Linux 지원)

  2. 번들 도구 다운로드, 여기서는 Rufus 예제를 사용할 것입니다. 실제로는 다른 도구도 사용 가능합니다, 예를 들어 Etcher, Win32 Disk Imager 등.

    • Rufus 다운로드 주소: 한글 다운로드 또는 영문 다운로드 (Windows 전용).
    • Etcher 다운로드 주소: 다운로드 (Windows, MacOS, Linux).
    • Win32 Disk Imager 다운로드 주소: 다운로드 (Windows 전용).

  3. 튜토리얼에서는 FlyOS-Fast-H5.exe를 예제로 사용할 것입니다.

  4. 시스템 이미지 파일 압축 해제, .img 파일 얻기.

    • FlyOS-Fast-H5.exe를 더블 클릭하거나 우클릭 후 열기를 선택하면 몇 초 후 아래와 같은 창이 나타납니다 (시스템에서 불안전하다고 경고하는 경우 "더 자세히 보기" -> "실행" 선택).
    • 압축 디렉토리를 선택한 후 추출 버튼을 클릭하고 해제 과정이 완료될 때까지 기다립니다.
    • 해제 작업이 완료되면 압축 디렉토리에 FlyOS-Fast-H5-vx.x.x.img 파일이 생성됩니다. x.x.x는 시스템 버전 번호입니다.

  5. 이미지 번들.

    • TF 카드에 번들하기
    • USB로 M2WE에 번들하기
    • M2SD 번들러로 M2WE 또는 TF 카드에 번들하기
      • 번들용 TF 카드는 4GB보다 큰 용량이 필요하며, 추천하는 제품은 SanDisk 32GB C10급 이상의 TF 카드입니다 (저렴한 카드들은 많은 경우 번들 후 정상적으로 실행되지 않을 수 있음).
      • 번들용 TF 카드는 SD 카드 리더기를 사용해야 하며, 번들 카드를 리더기에 삽입한 후 컴퓨터에 리더기를 연결합니다.
    2. Rufus를 열고 장치를 선택합니다. 선택된 장치의 표시 용량이 컴퓨터에 삽입된 번들 카드 용량과 일치해야 합니다.

    1. 이미지 파일 선택, 유형 선택 오른쪽의 선택 버튼을 클릭하여 압축해제된 .img 파일을 선택합니다.

    2. 이미지 파일을 선택한 후, 유형 선택이 이미지 파일의 이름으로 바뀝니다.

    3. 시작 버튼을 클릭하면, 데이터를 지우라는 경고 창이 나타나면 확인을 클릭합니다.

    4. 번들이 완료되면 컴퓨터가 경고음을 내며 진행률 표시줄이 준비 완료로 표시됩니다.

    5. 번들이 완료되면 바로 번들 카드를 빼지 마세요.

    6. 파일 탐색기를 열고 FlyOS-Conf라는 U디스크 또는 이동식 디스크가 존재하는지 확인합니다.

    7. FlyOS-Conf를 열면 아래 그림과 같은 두 개의 파일이 존재한다면 번들이 성공적으로 이루어졌습니다.

    8. config.txt는 시스템 구성 파일이며 다음 페이지의 내용을 참조하세요. 구성 파일 수정.

    1. 드라이버 다운로드:

      • M2WE 드라이버
      • M2WE BOOT 시작 도구

    2. 드라이버 설치:

      • Zadig를 열고 Options 메뉴에서 모든 장치 나열을 체크합니다
    1. M2WE를 설치한 후, 보드의 BOOT 버튼을 누른 후 양쪽 USB 케이블로 상위 기계를 컴퓨터에 연결합니다.

    1. zadig를 엽니다.

    2. 문제가 없다면, 이때 zadig에 알 수 없는 장치 #x 또는 USB 장치(VID_1f3a_PID_efe8)가 추가됩니다. 그것을 선택합니다.

    3. 장치를 선택한 후 USB ID가 위의 그림과 같았는지 확인합니다. 아니면 다른 장치를 선택합니다.

    4. 위의 그림에서 두 번째 부분은 WinUSB(vxxxxxxx)만 선택하면 됩니다. 버전 번호는 중요하지 않습니다.

    5. 드라이버 설치를 클릭하여 드라이버를 설치합니다. 같은 컴퓨터에서는 드라이버를 한번만 설치하면 됩니다

    6. 설치된 FLY-BOOT를 열고 장치 확인을 클릭하면, 앞에서 한 모든 작업이 정확히 이루어졌다면 "사용 가능한 장치 발견"이라는 메시지가 표시됩니다.

    7. BOOT 시작을 클릭하면 정상적으로 작동한다면 "BOOT 성공적으로 활성화"라는 메시지가 표시되고 몇 초 후 다음 단계로 넘어갈 수 있습니다.

    1. Rufus를 열고 장치를 선택합니다. 선택된 장치의 표시 용량이 컴퓨터에 삽입된 번들 카드 용량과 일치해야 합니다.

    2. 이미지 파일을 선택한 후, 유형 선택이 이미지 파일의 이름으로 바뀝니다.

    3. 시작 버튼을 클릭하면, 데이터를 지우라는 경고 창이 나타나면 확인을 클릭합니다.

    5. 번들이 완료되면 바로 번들 카드를 빼지 마세요.

    6. 파일 탐색기를 열고 FlyOS-Conf라는 U디스크 또는 이동식 디스크가 존재하는지 확인합니다.

    7. FlyOS-Conf를 열면 아래 그림과 같은 두 개의 파일이 존재한다면 번들이 성공적으로 이루어졌습니다.

    주의사항
    • 본 제품은 USB 3.0을 지원합니다.
    • 본 제품은 TF와 M2WE를 동시에 설치한 상태에서 함께 사용할 수 없습니다.
    • M2WE 또는 SD카드를 컴퓨터의 USB포트에 연결하면 포맷이 가능합니다.
    • NVMe 등 다른 장치를 설치하면 장치가 손상될 수 있으니 절대로 설치하지 마십시오!!

    1. M2WE 또는 SD카드를 M2SD에 설치합니다. TF카드를 사용하여 번들하는 경우, TF카드는 4GB보다 큰 용량이 필요하며, 추천하는 제품은 SanDisk 32GB C10급 이상의 TF카드입니다 (저렴한 카드들은 많은 경우 번들 후 정상적으로 실행되지 않을 수 있음).
