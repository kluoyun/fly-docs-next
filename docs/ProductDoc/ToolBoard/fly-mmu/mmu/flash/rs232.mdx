# RS232 固件编译

## 开始编译

    <SSH />

    <MakeOperate/>

    <RS232 />

    * 选择GPIO pins to set at micro-controller startup，输入：!PA15 请注意!是大写
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-mmu/mmu/flash/img/rs232.png').default} size="100%" align="center" />
    * 按 `Q` 键，出现 **Save configuration**，这时再按 `Y` 键
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * 现在应该保存了配置并且退出到了命令行界面
    * 输入 `make -j4` 开始编译，时间有点长
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * 最后输出下面的内容则编译成功

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```


## 进入烧录模式

    :::tip

    上方为`RESET按键`，下方为`BOOT按键`

    :::

    * 进入DFU有两种方法
    * 第一种：完全主板完全断电情况下，按着BOOT按键然后将带有数据传输的数据线一头接主板一头接上位机
    * 第二种：在将带有数据传输的数据线一头接主板一头接你上位机，按着BOOT键然后按下RST键，然后松开RST键最后松开BOOT键
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-mmu/mmu/flash/img/boot.png').default} size="100%" align="center" />

## 烧录固件 
    
    <Katapult />
    
    3. 烧录固件
    
        ```bash
        sudo dfu-util -a 0 -d 0483:df11 --dfuse-address 0x08000000 -D ~/kliper/out/klipper.bin
        ```

    4. 出现下图箭头处所示内容``File downloaded successfully``则烧录成功

        <ImageView image={require('@site/docs/General/dfu-flash/img/make-flash.png').default} size="100%" align="center" />

    :::tip

    烧录完成后，需要给主板彻底断电一次。

    :::
