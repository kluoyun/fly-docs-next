# Écran de démarrage

        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-uip/img/uip.webp').default} size="100%" align="left" />

## Première interface
    :::warning Remarques importantes

    * Le système doit prendre en charge ECM ou Rndis, sinon il ne pourra pas être utilisé normalement
    * Ce produit est conçu spécialement pour Klipper et nécessite la collaboration du service Moonraker pour afficher l'IP et les ressources ; dans le cas contraire, il faut activer le backend et configurer le mode SSH pour l'utiliser
    
    :::
    * L'affichage de l'IP provient des informations lues par la machine connectée ; si aucune IP n'est lue ou si elle n'est pas disponible, un message sera affiché
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-uip/img/ip.webp').default} size="100%" align="left" />
    * Si une IP est correctement lue, l'interface suivante apparaît (l'IP dépend de la machine supérieure)
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-uip/img/ip1.webp').default} size="100%" align="left" />

## Deuxième interface

    * Affiche l'utilisation du CPU et de la mémoire de la machine connectée
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-uip/img/cpu.webp').default} size="100%" align="left" />

## Troisième interface

    * Affiche les services démarrés sur la machine supérieure ; notez que les services affichés sont liés à Klipper
    * S'il n'est pas démarré, un point rouge apparaît devant le service
    * S'il est démarré, un point bleu apparaît devant le service
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-uip/img/service.webp').default} size="100%" align="left" />
