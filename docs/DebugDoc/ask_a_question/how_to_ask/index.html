提问前先阅读

  • QQ/微信等交流工具提供了快速便捷的交流方式，但如果遇到问题，建议遵循以下步骤。
  • 最终能否得到有用的回答，往往取决于你提问和追问的方式。

问题解决步骤：​

  1. 尝试自身解决：运用你掌握的知识先尝试自己解决问题。不要被英文输出信息吓倒，尽量读懂它。绝大多数信息输出都有指向性，你只需要找到它，然后自己尝试解决。
  2. 在网上搜索：在网上搜索一下，看看有没有人遇到过类似的问题。如果有，请先参考一下别人的解决方案，如果没有，再在群里提问。
  3. 在群里提问：在群里提问，请先描述清楚问题，然后附上截图，方便大家快速定位问题。你准备的信息越丰富，别人解决问题的效率就越高。

提问时，尽量做到以下几点，会大大提高问题被快速解决的几率：​

时刻站在回答者的角度思考问题​

每次提问，都要站在解答者的角度思考：

  • 假设解答者都是大佬，他们时间宝贵。
  • 如何让大家在一瞥之间就能看懂我的提问，最好是不要让回答者来追问我做了什么，目的是什么等基础问题。
  • 如果让大家愿意回答我的问题，比如是有技术讨论深度，还是说能让回答者感受到至高无上的被崇敬的感觉，还是说我能直接打钱。
  • 不要让回答者觉得我懒，没有看文档，没有看代码，没有看日志，没有搜索过，没有思考过，没有尝试过，没有做过任何努力，就直接来问问题。

缕清问题， 弄清楚到底发生了什么，以及我做了什么，包括：​

  • 我想实现什么效果，什么功能？
  • 为了实现这个效果，我是怎么做的， 详细的过程是什么？
  • 实施的过程中， 什么错误发生了， 现象是什么（比如报错了， 报了什么错，完整的错误内容是什么？切忌只看最后几行，尽量往前多看看日志）
  • 我是否认真看了报错的信息，报错信息中是否有提示了错误的原因和解决的方法？
  • 根据这些报错信息，仔细思考，是否能解决问题？
  • 搜索 文档、 FAQ， issue， 以及用搜索引擎能不能搜索到问题的解决方法
  • 如果是感觉文档哪里不完事，可以直接指出来！！不要担心其他，我不是大佬肯定有考虑不到位的地方！！

低效甚至无效提问​

  • 问：请问大佬，这个问题怎么解决？

存在问题​

  • 随手拍屏 能截图就不要拍屏，请学会使用各种工具附带的截图功能。除非特殊情况经历拍清晰一点
  • 信息不完全 CAN找不到ID问题有很多种如，线接错，上位机与下位机速率等等将您的问题限定在特定场景中，给出具体条件，有助于聚焦问题。
  • 不会截屏情况这个截屏教程

提问模板​

提问尽量做到提问优雅，不需要添加多余的语气词，抱怨的词汇，斟酌每一个词和标点符号，站在回答人的角度思考问题，如何让回答的人快速帮助自己解决问题， 字数太少描述不清楚， 字数太多则让人没有耐心

如果问题自己实在不能解决， 需要请教于人， 需要考虑：​

  • 请教谁，在哪里请教， 谁会更有几率回答我的问题？以及实时性如何？

  • 我应该给他提供什么信息和现象他才能愿意快速帮我解决问题？

    • 提供我的目的 （方便让回答者知道你在做什么事情）
    • 提供完整的实施过程，以及过程中出现的现象 （方便回答者跟着你的流程去做一遍， 即问题复现）
    • 给出错误的地方，指明哪里出现的现象或结果和自己预期的不一样！ （让回答者知道，哪里没达到预期）
    • 提供出现的错误信息，需要完整，尽可能更多地截图，更多的日志，千万别小气地截一张小图，或者之给出日志的一部分（ 因为回答者可能很久没做这件事情了，忘记了一些细节，需要靠截图和完整日志来快速回忆起来; 而且根据详细的日志可以快速定位到问题处在哪里）

  • 怎么提问态度显得比较诚恳，就算我再小白，大家也愿意回答

标题​

无论在哪里提问给自己的问题拟定一个30个字左右标题， 阐明问题的中心思想，包括：

  • 问题类别， 是问题请教，还是 BUG提交，还是经验分享等等。让大家在满是文字的屏幕上可以立马定位到你想做什么
  • 一句话阐明问题的中心你思想，比如遇到 CAN找不到ID，主板连接不上等等

所以综合后标题可以是这样：

  • 【提问】 SHT36V3无法找到CANID已经确认过什么什么

这样的标题一定不能出现:

  • 啊啊啊啊啊为什么我的板子找不到ID（我咋知道，你怎么接线和操作的我又不清楚）
  • 为什么我的上位机不启动（我咋知道，什么上位机都不清楚）
  • 为什么我这没有设备啊 （我咋知道，啥产品都不知道）
  • 这个报错是什么错啊 （没读文档，一来开问）
  • 收到了主板，驱动怎么接？ （没读文档，一来开问）
  • 这个线怎么接 （什么问题呢）
  • 为什么进去DFU后烧录不了，为什么当我进不去DFU等等 (你得告诉大家你怎么操作，怎么接线，不然别人可能就是认为你在瞎说懒得理你)

可以这样问：

  • 【提问】我的板子在我接反电源后，现在板子无法启动，怎么判断板子哪里烧毁了，如果可以，怎么救活呢
  • 【BUG】这个文档此处说明不全，希望可以添加或者修改类似内容

参考模板​

  • 请花一点时间整理问题，描述清楚以下信息：
  • 最终能否得到有用的回答，往往取决于你提问和追问的方式。
  1. 什么问题 例如： 新买的主板与工具版找不到设备
  2. 什么硬件设备 例如： 机顶盒+D5主板+SHT36V3
  3. 怎么接线 例如： 接线拍高清图发给大佬
  4. 什么问题现象 例如: 上位机输入lsusb没有设备等等。附完整信息截图。

【提问】我的买了一张D5主板与SHT36在连接机顶盒上位机使用lsusb无法找到任何设备
