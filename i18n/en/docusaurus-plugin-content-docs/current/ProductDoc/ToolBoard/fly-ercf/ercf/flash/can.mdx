# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <CAN />

    * Choose GPIO pins to set at micro-controller startup, input: gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/can.webp').default} size="100%" align="center" />
    * Press `Q` key, the prompt **Save configuration** appears, then press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you should be back to the command line interface
    * Input `make -j4` to start compilation, it takes some time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * The final output should look like this for a successful compilation:

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flash Firmware

    * Before flashing the firmware, connect the tool board to UTOC or a mainboard flashed with bridge firmware using an XT30 cable. Below is a reference wiring diagram.
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Ensure that the CAN H and CAN L have a resistance of around `60立` when measured with a multimeter in a power-off state. If it reads `120立`, connect a `120立` jumper. If it reads around `40立`, remove the jumper.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/120.webp').default} size="50%" align="center" />
