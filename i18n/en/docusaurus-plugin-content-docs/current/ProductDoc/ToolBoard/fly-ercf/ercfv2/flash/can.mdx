    * Select GPIO pins to set at micro-controller startup, input: gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/can.png').default} size="100%" align="center" />
    * Press `Q` key, then you will see **Save configuration**, press `Y` key again
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Now the configuration should be saved and you are back to the command line interface
    * Input `make -j4` to start compilation, it may take a while
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * The compilation is successful if the following output is displayed

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flash Firmware

    * Before flashing the firmware, use an XT30 cable to connect the tool board to UTOC or the motherboard with bridge firmware flashed. Below is a reference wiring diagram.
    * Please check if the DIP switches are set correctly
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Ensure that the resistance between CAN H and CAN L is around `60立` when measured with a multimeter in a power-off state. If it is `120立`, please connect a `120立` jumper wire. If it is around `40立`, please remove the jumper wire
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/120.png').default} size="100%" align="center" />
