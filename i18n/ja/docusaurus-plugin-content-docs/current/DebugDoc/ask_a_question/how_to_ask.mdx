# 質問する前にお読みください

- QQ/WeChat などのコミュニケーション ツールは、迅速で便利なコミュニケーション方法を提供しますが、問題が発生した場合は、次の手順に従うことをお勧めします。
- 最終的に有益な答えが得られるかどうかは、多くの場合、質問とフォローアップの方法によって異なります。

## 問題解決の手順:

1. 自分で問題を解決してみる: 持っている知識を活用して、まず自分で問題を解決してみます。英語のエラー出力に怖がらずに、読んでみてください。画面に表示されているエラーメッセージ内容を確認することでおおよその問題は解決できます。必要なのは内容を確認して自分で解決しようとすることだけです。
2. ウェブを検索する：同じような問題に遭遇した人がいないか、ウェブを検索してください。 もし同じ症状が見つかった場合、まず他の人の解決策を参考にし、そうでなければグループで質問してください。Reddit/Facebook/X/Discordなども非常に役に立ちます。
3. グループ内で質問する：グループ内で質問する場合、まず問題を明確に説明し、次にスクリーンショットを添付してください。 より多くの情報を準備すればするほど、他の人はより効率的に問題を解決することができます。

## 質問をするときは、次のことを心がけると、質問が早く解ける可能性がぐっと高まります：

### 常に回答者の立場に立って考えてみてください

質問するときは常に、回答者の視点から考えてください。:

- 回答者は全員偉い人で、彼らの時間は貴重であると仮定します。
- 私の質問を一目で誰もが理解するにはどうすればよいでしょうか? 回答者に私が何をしたのか、私の目的は何なのかなどの基本的な質問をさせないのが最善です。
- 私の質問に答えたくなるような、技術的な議論が深まるような、回答者が至高の存在として尊敬されるような、あるいはお金になるようなことがあれば素敵ですね。
- 回答者に、私が怠け者で、ドキュメントを読まず、コードを見ず、ログを見ず、検索せず、考えず、試さず、何の努力もせず、ただ質問に直球で来たと思われないように心がけましょう。

### 何が起きたのか、そして問題が起きた際に何をしたのかを正確に把握し、問題をまとめる：

- どのような効果を、どのような機能を実現したいのか？
- その効果を得るために何をしたのか、細かいプロセスはどうだったのか。
- 実際の過程で、どのようなエラーが発生したのか、どのような現象なのか（例えば、エラーが報告された、どのようなエラーが報告されたのか、**完了**エラーの内容は何か。 最後の数行だけを見るのではなく、できるだけログを見るようにしてください）
- エラーを報告するメッセージを注意深く見たか、エラーメッセージにエラーの原因や解決方法のヒントがあったか。
- これらのエラーメッセージをもとに、問題が解決できるかどうかをよく考えてください。
- ドキュメント、FAQ、問題を検索し、問題の解決策を見つけるために検索エンジンを使用したかどうか。
- `ドキュメントで網羅していない点を見つけたら、それを指摘してください！心配しないでください......人間なので間違えることはあります！`

## 非効率的な質問、または非効果的な質問

- 質問: 先生、この問題はどうやって解決しますか?

### 問題の探求

- *画面をスマホで撮影* できればスマホで画面を撮影せず、例えばパソコンに付属しているスクリーンショット機能の使い方を覚えてください。 特別な状況でない限りはスクショを使用し、どうしても使えない場合は鮮明に撮影しましょう。
- *不完全な情報* CANでIDが見つからない問題には多くの原因があります。例えば、配線ミス、上位機と下位機の速度の違いなどです。問題を特定のシナリオに限定し、具体的な条件を提示することで、問題の焦点を絞ることができます。
- スクリーンショットの撮り方がわからない場合は、スクリーンショットの撮り方に関するこの[チュートリアル](https://www.bilibili.com/video/BV1LB4y1t7jy/?spm_id_from=333.337.search-card.all.click&vd_source=36affb37d7228751cd1fcbbda487dfdf)をご覧ください。

## 質問テンプレート

質問はできるだけ優雅に行い、余計な語気や不満の言葉を加えないようにしましょう。言葉や句読点を慎重に選び、回答者の立場に立って、どのようにすれば回答者が迅速に問題を解決できるかを考えます。文字数が少なすぎると説明が不十分になり、文字数が多すぎると読むのが面倒になります。

### もし自分で問題を解決できない場合、他人に助けを求める際には以下の点を考慮する必要があります。：

- 誰に、どこで質問するか、誰が私の質問に答える可能性が高いか、そしてリアルタイム性はどうか？

- 回答者が迅速に問題を解決するために、どのような情報や現象を提供すればよいでしょうか？

    - 自分の目的を提示する（回答者に自分が何をしているのかわかりやすくするため）
    - 完全な実施プロセスとその過程で発生した現象を提供してください（回答者があなたの手順に従って再現できるようにするため）
    - エラーを指摘し、実際の状況や結果が期待とどこで異なるかを指摘してください！（回答者にどこが本来期待していた動作・結果に達していないかを知らせてください）
    - エラー情報を提供する際には、できるだけ多くのスクリーンショットやログを含めて、完全な情報を提供してください。小さなスクリーンショットや部分的なログだけを提供するのは避けてください。回答者が長い間その作業をしていない場合、詳細なスクリーンショットや完全なログが必要です。これにより、迅速に問題の場所を特定し、記憶を呼び起こすのに役立ちます。

- どうすれば質問者の態度が誠実に見え、初心者でも皆が喜んで答えてくれるでしょうか？

### タイトル

どこで質問する場合でも、自分の質問に**30文字**程度のタイトルを付け、問題の中心思想を明確にしてください。具体的には：

- 質問の種類を明確にしてください。例えば、質問の内容が問題の相談なのか、バグの報告なのか、経験の共有なのかなどです。これにより、他の人が大量のテキストの中からあなたの意図をすぐに理解できるようになります。
- 問題の要点を一言で説明してください。例えば、**CANがIDを見つけられない、マザーボードが接続できない**などです。

したがって、総合的なタイトルは次のようになります：

- `【MaixPy 質問】SHT36V3がCANIDを見つけられない、すでに確認済みの内容`

このようなタイトルは**絶対に**避けてください：

- `ああああああ、なぜ私のボードがIDを見つけられないのか`（どうやって配線や操作をしたのか分からないので、私には分かりません）
- `なぜ私の上位機が起動しないのか`（どうやって配線や操作をしたのか分からないので、私には分かりません）
- `なぜ私のところにデバイスがないのか`（どうやって配線や操作をしたのか分からないので、私には分かりません）
- `このエラーは何のエラーですか`（ドキュメントを読まずに、いきなり質問）
- `マザーボードを受け取りましたが、ドライバはどう接続しますか？`（ドキュメントを読まずに、いきなり質問）
- `このケーブルはどう接続しますか`（どのような問題ですか）
- `なぜDFUモードに入った後に書き込みができないのか、なぜDFUモードに入れないのかなど`（操作方法や接続方法を教えてくれないと、他の人はあなたが適当に言っていると思って無視するかもしれません）

次のように尋ねることができます。

- `【MaixPy 質問】私のボードは電源を逆に接続した後、現在ボードが起動しません。どこが焼損したかを判断する方法と、可能であれば修復方法を教えてください。`
- `【MaixPy BUG】このドキュメントのこの部分の説明が不完全です。類似の内容を追加または修正していただけると助かります。`

### 参照テンプレート

- 少し時間をとって質問を整理し、次の情報を明確に説明してください。
- 最終的に有益な答えが得られるかどうかは、多くの場合、質問とフォローアップの方法によって異なります。

1. **問題は何ですか?**たとえば、新しく購入したマザーボードとツールボードがデバイスを見つけられません。
2. **どのようなハードウェア機器ですか**? 例: セットトップ ボックス + D5マザーボード + SHT36V3
3. **配線の接続方法**例: 配線の高解像度写真を撮り、送信してください。
4. **問題現象は何ですか** 例：上位入力にデバイスがない<br>**lsusb** など、メッセージのスクリーンショットを添付します。

【MaixPy による質問】 D5 マザーボードと SHT36 を **lsusb** を使ってセットトップボックスのホストコンピュータに接続したのですが、デバイスが見つかりません。
