进入烧录模式​

  • 进入烧录模式有两种方法
  • 第一种：主板完全断电情况下，短接BOOT和BOOT1然后将带有数据传输的数据线一头接主板一头接你设备
  • 第二种：在将带有数据传输的数据线一头接主板一头接你设备后，短接短接BOOT和BOOT1然后按下RST键，然后松开RST键最后断开短接BOOT跳线

烧录固件​

  • windows烧录教程
  • 上位机烧录教程

  • Gemini的bootloader固件

    点击自动下载固件

  • 拨码开关3,4往右边拨，将下位机与上位机通过内置USB连接

  • 拨码开关1,2往左边拨，否则会影响下位机与上位机连接

须知
  • 系统兼容性问题，请使用电脑的USB2.0烧录
  • 如果还是烧录不了请换上位机进行烧录
  1. 烧录工具 下载地址
  2. 需要下载SetupSTM32CubeProgrammer与STM32 DFU驱动
  3. 将前面编译好的固件(klipper.bin)或者预先编译好的Katapult固件复制到电脑任意目录
  4. 使用Type-C数据线将主板连接到电脑，请确保已经进入DFU烧录模式
  5. STM32CubeProgrammer中选择USB模式，并刷新，连接
  6. 如果没有出现错误弹窗则连接成功
  7. 点击左边下载图标并且将下方几个选择上
  8. 点击箭头所指的Browse，然后选择你需要烧录的固件
  9. 选择Start address输入0x08000000
  10. 点击下方的Start automatic mode开始烧录固件
  11. 出现下方提示则代表固件烧录完成
  • 拨码开关1,2往右边拨，将下位机与上位机通过内置USB连接
  • 拨码开关3,4往左边拨，否则会影响下位机与上位机连接
  • 按着此方法拨动拨码开关后，下方usb口将被下位机占用，请不要接入其他设备
  1. 安装烧录工具
  • FLY上位机无需执行安装烧录工具
  • 如果你需要安装请注意：上位机需要能上网才可以正常安装
  • 如果已经安装过则无需安装
sudo apt update
sudo apt install dfu-util -y
  1. 执行下面的命令查看是否连接成功。
lsusb
  • 如果没有DFU设备请尝试重新进入烧录模式
  • 因系统版本问题可能显示其他名字，如果有0483:df11则代表进入烧录模式
  1. 拉取预先编译固件
  • 这里面都是预先编译好的固件
  • 请注意上位机需要能上网才可以安装
  • 上位机需要执行下方命令下载固件包
cd && git clone https://github.com/FengHua741/FLY_Katapult.git
  • 如果已经下载过固件包的建议更新固件包后在进行剩下操作
cd ~/FLY_Katapult && git pull
  1. 烧录固件
ヒント

只需要烧录其中一个固件

  • 下方是bootloader固件刷写指令 
    sudo dfu-util -a 0 -d 0483:df11 --dfuse-address 0x08000000 -D ~/FLY_Katapult/BL/Gemini/GeminiV3.bin
  1. 出现下图箭头处所示内容File downloaded successfully则烧录成功
手記

ヒント
