FlyOS-Fast よくある質問

質問：コマンドを実行すると「command not found」と表示される​

  • 原因：システムにそのコマンドが存在しません。
  • 解決策：システムには多くの一般的なソフトウェアパッケージが含まれています。必要な他のパッケージがある場合、私たちにリクエストを送信してください。必要に応じて、次のバージョンで事前にインストールされます。

質問：なぜapt updateコマンドが実行できないのですか？​

  • 原因：FlyOS-Fastは軽量化された高速システムであり、aptパッケージ管理ツールがありません。そのため、apt updateコマンドを実行できません。
  • 解決策：システムには多くの一般的なソフトウェアパッケージが含まれています。必要な他のパッケージがある場合、私たちにリクエストを送信してください。必要に応じて、次のバージョンで事前にインストールされます。

質問：焼付けコマンドを実行すると「-bash: ~/klippy-env/bin/python: No such file or directory」というエラーが出る​

  • 原因：FlyOS-Fastはvenv仮想環境を使用していません。
  • 解決策：~/klippy-env/bin/pythonをpythonに置き換えて実行してください。

質問：なぜflyユーザーにログインできないのですか？​

  • 原因：FlyOS-Fastにはflyユーザーが存在せず、すべての操作はrootユーザーで行われます。
  • 解決策：システムにはflyユーザーが存在しないため、直接rootユーザーでログインしてください。

質問：なぜ9999ポートのfly-toolsにアクセスできないのですか？​

  • 原因：FlyOS-Fastにはfly-toolsがインストールされていません。
  • 解決策：fluiddとmainsailは既にIDの照会をサポートしています。スクリーンなどの設定はシステム構成にあります。

質問：なぜ画面にLOGOしか表示されず、KlipperScreenインターフェースが表示されないのですか？​

  • 原因：KlipperScreenが有効になっていません。
  • 解決策：システム構成でKlipperScreenを有効にし、klipper_screenを1に設定してください。

質問：なぜシステムはWebページからklipperなどのコンポーネントを更新しませんか？​

  • 原因：FlyOS-Fastに含まれるklipper、moonrakerなどのコンポーネントは二次開発バージョンです。
  • 解決策：必要に応じて、コマンドラインで更新できますが、推奨されません。システムの更新をお待ちください。

質問：Happy Hareなどのサードパーティ製コンポーネントのインストールスクリプトが使用できない？​

  • 原因：FlyOS-Fastのデフォルトユーザーはrootであり、これらのサードパーティのインストールスクリプトはrootとして認識され、インストールされません。
  • 解決策：簡単な修正でインストールスクリプトのroot検出をバイパスできます。詳細はこちらを参照してください。他のインストールスクリプトも同様です。
