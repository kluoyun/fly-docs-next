CAN 펌웨어 컴파일

컴파일 시작​

  • klipper 서비스가 설치된 상위 기기를 사용해야 합니다 네트워크를 통해 SSH 연결

    • 그리고 장치를 상위 기기와 연결해야 합니다
    • 주의: 일반적인 상위 기기를 사용하세요, 예를 들어 라즈베리 파이, FLY파이 등, 만약 커스텀 상위 기기라면 문제가 너무 많아서 여기서는 많은 도움을 제공할 수 없습니다
    주의사항
    • ROOT 사용자를 사용하지 마세요
팁
  • 请确保上位机可以正常联网
  • 固件编译前需要使用SSH通过网络登录上位机
  • 请不要使用串口工具登录上位机

Klipper固件编译配置页面操作说明​

팁

请确保键盘输入法为半角模式，即英文模式

  1. 键盘上键↑，下键↓，用于上下移动光标来选中菜单项
  2. 确认键Enter 或 空格键Space，用于勾选菜单或进入子菜单
  3. 退出键ESC，用于返回上一级菜单
  4. Q键，用于退出Klipper固件配置页面
  5. Y键，在退出Klipper固件配置页面时，如果有提示，则点击Y键保存配置文件
경고

如果配置页面选项比较少，请先勾选[ ] Enable extra low-level configuration options，用于显示部分隐藏选项

다음은 펌웨어를 컴파일하는 방법을 소개합니다:

  • SSH에 연결한 후 아래 명령을 입력하고 엔터를 누릅니다:

    cd ~/klipper && rm -rf ~/klipper/.config && rm -rf ~/klipper/out && make menuconfig

  • 여기서 rm -rf ~/klipper/.config && rm -rf ~/klipper/out는 이전 컴파일 데이터와 펌웨어를 삭제하는 것이고 make menuconfig는 펌웨어를 컴파일하는 것입니다. 실행 후 아래와 같은 화면이 나타나야 합니다.

  • Enable extra low-level configuration options을 선택하고 엔터를 누릅니다.

  • 메뉴 Micro-controller Architecture로 들어가 STMicroelectronics STM32를 선택하고 엔터를 누릅니다.

  • 메뉴 Processor model로 들어가 STM32F072를 선택하고 엔터를 누릅니다.

  • Bootloader offset을 선택하고 8KiB bootloader를 선택합니다.

  • Communication interface를 선택하고 **CAN bus (on PB8/PB9)**를 선택합니다.

  • Q 키를 누르면 Save configuration이 나타나고, 이때 Y 키를 다시 누릅니다

  • 이제 설정이 저장되고 명령 줄 인터페이스로 돌아갔어야 합니다

  • make -j4를 입력하여 컴파일을 시작합니다. 시간이 좀 걸립니다

  • 마지막으로 다음과 같은 내용이 출력되면 컴파일이 성공한 것입니다

    Linking out/klipper.elf
    Creating hex file out/klipper.bin

장치 검색​

  • 상위 컴퓨터가 정상적으로 네트워크에 연결되었는지 확인하세요

    • 하위 컴퓨터가 상위 컴퓨터와 데이터 전송 기능을 가진 데이터 케이블로 연결되었는지 확인하세요
    • SSH 연결을 통해 상위 컴퓨터에 접속한 후 lsusb를 입력하고 엔터를 누르면 다음 세 가지 상황이 발생할 수 있습니다
      • 인식된 USB 장비가 표시됩니다
      • ls 명령어가 없다는 메시지가 표시되면, 다음 명령어를 실행하여 설치할 수 있습니다 
        sudo apt-get install usbutils
      • 오류 메시지 없이 다른 정보가 없으면 (시스템 문제로 인해 시스템을 교체해야 합니다)
    • lsusb를 입력한 후 반응이 없으면, 이는 시스템 문제로 인해 여기서는 해결할 수 없으며 정상적인 시스템으로 교체해야 합니다
    • 아래 이미지에서 원 안에 표시된 정보가 나타나면 다음 단계로 진행할 수 있습니다
    • 1d50:606f는 이번에 사용할 장비에 속합니다. 후속 메시지는 신경 쓸 필요가 없습니다. 시스템 문제로 인해 완전히 표시되지 않거나 아예 표시되지 않을 수 있습니다
    • 두 개의 1d50:606f가 있으면, 왜 두 개가 있는지 확인하고 그 중 하나를 제거한 후 시스템을 재부팅해야 합니다
    • 두 개의 1d50:606f는 후속 펌웨어와의 연결에 영향을 미칠 수 있으며, 예를 들어 FLY MINI PAD의 경우, 다른 CAN 브리징 장비를 사용하지 않고 내장 UTOC만 사용하는 것이 좋습니다
    • 1d50:606f가 없으면, 데이터 케이블이 제대로 연결되었는지, 펌웨어가 제대로 설치되었는지 스스로 확인하세요
    주의 사항

    1d50:606f가 있을 때 다음 단계로 진행할 수 있습니다

펌웨어 플래싱​

  • 펌웨어를 플래싱하기 전에 XT30 데이터 케이블을 사용하여 도구 보드를 UTOC 또는 브리징 펌웨어가 플래시된 메인보드에 연결합니다. 아래는 참고 회로도입니다
  • CAN H와 CAN L이 전원이 꺼진 상태에서 멀티미터로 측정한 저항 값이 60Ω 정도 되는지 확인하십시오. 120Ω일 경우 120Ω 점퍼를 연결해야 하며, 40Ω 정도일 경우 점퍼를 제거해야 합니다
팁
  • ID 찾기와 펌웨어 업데이트는 인쇄 중에 수행하지 않는 것이 좋습니다.
  • 동일한 메인보드 또는 툴보드의 Katapult CAN 펌웨어와 Klipper CAN 펌웨어는 ID가 완전히 동일합니다.
경고
  • 상위기기의 CAN 네트워크가 이미 설정되었는지 확인하십시오.
  • lsusb 명령 후 1d50:606f가 있을 때만 아래 명령으로 CAN ID를 검색할 수 있습니다.
  • 툴보드 CAN 속도는 상위기기와 동일해야 하며, 그렇지 않으면 ID를 찾을 수 없고 펌웨어를 플래시할 수 없습니다.

  • 아래 명령을 실행하여 CANBus UUID를 조회하십시오:

    ~/klippy-env/bin/python ~/klipper/scripts/canbus_query.py can0

    • 입력 후 기기에 따라 다음과 같은 상황이 발생할 수 있습니다, 참고용입니다:

      • 첫 번째: 브리지 펌웨어가 설치된 메인보드, 하나의 ID가 나타나고 Application: Klipper라고 표시됩니다.
      • 두 번째: 브리지 펌웨어가 설치된 메인보드와 펌웨어를 플래시해야 하는 툴보드, 두 개의 ID가 나타납니다.
      • 세 번째: UTOC와 함께 사용되는 툴보드를 플래시해야 하는 경우
      • 참고: UTOC는 ID가 표시되지 않습니다.
      • 네 번째: ID가 표시되지 않음, 일반적으로 Klipper가 점유 중이거나 상위기기 CAN0 속도가 잘못되었거나 연결이 비정상적인 경우입니다.

    • 정상적인 경우, 플래시해야 하는 툴보드가 CANBOOT 상태에 있을 때 Application: CanBoot라고 표시되고 해당 펌웨어 램프가 깜빡입니다. 이 시점에서 Klipper 펌웨어를 플래시하지 않으면 연결할 수 없습니다!!!

    • 참고: Klipper 펌웨어가 이미 ID를 설정 파일에 입력한 경우, ID는 표시되지 않으며 Klipper에 의해 점유됩니다. 해당 ID를 차단한 후 메인보드의 전원을 완전히 끄고 다시 시작해야 합니다.

    경고
    • UUID가 조회되지 않으면 아래 문서를 참조하십시오
    • CAN 문제 모음

  • 펌웨어 플래시

    • 아래 명령의 241696050c56를 조회한 UUID로 교체하십시오, -u 뒤에 공백이 있음을 주의하세요

      python3 ~/klipper/lib/canboot/flash_can.py -u 241696050c56

    • 아래 그림과 같이 CAN Flash Success가 나타나면 플래시가 성공적으로 완료된 것입니다.

  • 펌웨어를 업데이트하려면 펌웨어를 컴파일한 후 플래시 명령을 다시 실행하면 됩니다. 하지만 ID를 찾거나 Klipper가 이미 연결되었는지 확인하십시오.

    python3 ~/klipper/lib/canboot/flash_can.py -u  <MCU ID>
