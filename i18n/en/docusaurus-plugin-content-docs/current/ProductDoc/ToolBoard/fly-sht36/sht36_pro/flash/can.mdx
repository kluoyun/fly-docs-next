# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, input: !gpio13 Please note ! is uppercase
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.png').default} size="100%" align="center" />
    * Press `Q` key, you will see **Save configuration**, then press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Now the configuration should be saved and you are back to the command line interface
    * Input `make -j4` to start compilation, it takes a while
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * The following output indicates successful compilation

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Burn Firmware

    * Before burning the firmware, use an XT30 data cable to connect the tool board to UTOC or a mainboard flashed with bridge firmware. Below is a reference wiring diagram.
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Ensure that the resistance between CAN H and CAN L is around `60立` in a power-off state. If it's `120立`, you need to connect a `120立` jumper. If it's around `40立`, remove the jumper.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.png').default} size="100%" align="center" />
