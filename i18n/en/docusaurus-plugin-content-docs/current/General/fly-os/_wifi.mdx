

    :::danger Notice

    * The FLY host machine needs to use the M2WE or 2.4G WIFI module, otherwise use WIFI.
    * Please note that Chinese name WiFi and dual-band (2.4G and 5G) WiFi are not supported.
    * If you are using mini pad or lite2, please ignore this warning.

    :::

## Connect to Host Machine

    * If configuring WIFI under BOOT disk fails, you can refer to this configuration for WIFI connection. Please note that serial port connection to the host machine is required.
    * First, connect to SSH via serial port [here](/docs/DebugDoc/BasicTutorial/index.mdx "Click to jump").
    * Execute the following command, you may be prompted to enter a password. Enter the password `mellow` and press Enter. Note that the password is invisible.

    ```cfg
    sudo nmtui
    ```

        <ImageView image={require('@site/docs/General/fly-os/img/nmtui/nmtui1.png').default} size="80%" align="left" />

        <ImageView image={require('@site/docs/General/fly-os/img/nmtui/nmtui2.png').default} size="80%" align="left" />

    * Select `Activate a connection` and press Enter.

        <ImageView image={require('@site/docs/General/fly-os/img/nmtui/nmtui3.png').default} size="80%" align="left" />

    * Enter the following interface, use the `↑ ↓` keys on the keyboard to select the WiFi you want to connect, press Enter, input the password, and wait for the connection to succeed.

        <ImageView image={require('@site/docs/General/fly-os/img/nmtui/nmtui4.png').default} size="80%" align="left" />

        <ImageView image={require('@site/docs/General/fly-os/img/nmtui/nmtui5.png').default} size="80%" align="left" />

    * After successful connection, it will look like the following image.

        <ImageView image={require('@site/docs/General/fly-os/img/nmtui/nmtui6.png').default} size="80%" align="left" />

    * Press ESC to exit and return to the command line interface, then input `ip a`. In the image, `192.168.2.126` is the IP address of the host machine.
