## Поиск USB ID
    1. В SSH введите команду `lsusb`, нажмите клавишу `Enter`. Если появляется `1d50:614e`, как показано на рисунке ниже, можно переходить к следующему шагу.
    <ImageView image={require('@site/docs/General/get-id/img/614e.webp').default} size="80%" align="center" />

    2. `1d50:614e` принадлежит устройству, которое вы используете.  
        Если команда `lsusb` показывает устройство, но не отображает `1d50:614e`, проверьте правильность режима переключателей на материнской плате, попробуйте пересобрать и записатьfirmware USB, а также попробуйте подключить к материнской плате питание 24В.

    :::warning Внимание
       Действия можно продолжать только после того, как будет найдено устройство с `1d50:614e`.
    :::

    3. В SSH введите следующую команду, нажмите клавишу `Enter`. Если все выполнено корректно, то появится строка с `синим ID`.
    ```bash
    ls /dev/serial/by-id/*
    ```
    :::warning Внимание
    - Приведенный на изображении `ID` является примером, каждый материнская плата имеет уникальный `ID`. Используйте фактически полученный `ID`.  
    - Если подключены несколько материнских плат, одновременно будут отображаться несколько ID. Выберите подходящий ID в соответствии с моделью `MCU` вашей материнской платы.
    :::

## Заполнение USB ID

    1. Откройте веб-интерфейс верхнего уровня, найдите раздел конфигурации в боковой панели слева:

    <table>
      <tr>
        <td>
          - Для fluidd: `{…}`, щелкните для доступа к `printer.cfg`
          <ImageView image={require('@site/docs/General/get-id/img/fluidd-cfg.webp').default} size="100%" align="center" />
        </td>
        <td>
          - Для mainsail: `Машина`, щелкните для доступа к `printer.cfg`
          <ImageView image={require('@site/docs/General/get-id/img/mainsail-cfg.webp').default} size="100%" align="center" />
        </td>
      </tr>
    </table>

    2. Скопируйте синий ID и найдите в файле `printer.cfg` опцию `[mcu]`. После параметра `serial:` укажите полученный `ID`, сохраните изменения и перезагрузите систему, чтобы подключиться к материнской плате.
    :::danger
    * Примеры ID внизу仅供参考! Пожалуйста, строго придерживайтесь формата, показанного на изображении ниже
    * Формат USB ID материнской платы:
     ```
    [mcu]  
    serial: /dev/serial/by-id/usb-klipper_******_**********-****
    ```
    :::

    <ImageView image={require('@site/docs/General/get-id/img/723id.webp').default} size="100%" align="center" />
    
    3. Если Klipper выводит сообщение об ошибке `ADC out of range`, это нормальное явление. Подключите нагревательный элемент и термодатчик к материнской плате и правильно настройте контакты датчиков для головки и нагревательного стола, затем сохраните и перезагрузите систему.

:::danger
Внимание: Все ID в документации являются примерами, на каждой материнской плате ID различаются. Пожалуйста, используйте фактически полученные ID.
:::
