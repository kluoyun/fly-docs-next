固件编译与烧录

准备工作​

  • 开始获取ID前，请先连接到SSH
  • 请确保下位机连接上位机的数据线具有数据传输功能
  • SSH连接连接到上位机后输入lsusb并且回车，将可能出现下面几种设备
  • 如果系统不支持lsusb可以执行下方指令进行安装
  • 如果lsusb后没有设备，请更换上位机！

编译Klipper固件​

  • Klipper固件编译配置页面操作说明
подсказка

请确保键盘输入法为半角模式，即英文模式

  1. 键盘上键↑，下键↓，用于上下移动光标来选中菜单项
  2. 确认键Enter 或 空格键Space，用于勾选菜单或进入子菜单
  3. 退出键ESC，用于返回上一级菜单
  4. Q键，用于退出Klipper固件配置页面
  5. Y键，在退出Klipper固件配置页面时，如果有提示，则点击Y键保存配置文件
warning

如果配置页面选项比较少，请先勾选[ ] Enable extra low-level configuration options，用于显示部分隐藏选项

下面介绍如何编译固件：

  • 连接到SSH后输入下面的命令并回车：

    cd ~/klipper && rm -rf ~/klipper/.config && rm -rf ~/klipper/out && make menuconfig

  • 其中 rm -rf ~/klipper/.config && rm -rf ~/klipper/out是删除之前编译数据与固件make menuconfig是编译固件,执行后应该出现下方界面

  • 选择Enable extra low-level configuration options并且回车

  • 进入菜单Micro-controller Architecture然后选择STMicroelectronics STM32回车

  • 进入菜单Processor model，选择STM32F407回车

  • 选择Bootloader offset，选择：32KiB bootloader

  • Communication interface，选择：USB (on PA11/PA12)

  • 按 Q 键，出现 Save configuration，这时再按 Y 键

  • 现在应该保存了配置并且退出到了命令行界面

  • 输入 make -j4 开始编译，时间有点长

  • 最后输出下面的内容则编译成功

    Linking out/klipper.elf
    Creating hex file out/klipper.bin

烧录固件​

须知
  • F404ZG预刷的是TF卡烧录的BL固件

准备工作​

  • 开始烧录固件前，请先使用网络连接SSH
  • 需要准备一张容量不超过32G的TF卡，并且建议TF卡的传输速度必须为class10级或class10级以上，建议使用闪迪等品牌的TF卡

下载固件​

  1. 使用之前准备好的TF卡

  2. 使用SDFormatter格式化

  3. 使用MobaXterm_Personal左边文件目录找到klipper选择然后找到out打开

  4. 并且将klipper.bin复制到TF卡里面，并且重命名为firmware.bin

  5. 主板断电，将TF卡插入主板

  6. 主板上，等待10秒

  7. 取下TF卡，插入电脑如果SD卡中的看firmware.bin消失，出现FLY.CUR就是烧录成功了

RP2040烧录说明​

  • 如果是不带偏移的RP2040固件则应该为klipper.uf2，可以直接放到RP2040的BOOT盘下，正常情况下会直接消失，代表固件已经烧录完成
