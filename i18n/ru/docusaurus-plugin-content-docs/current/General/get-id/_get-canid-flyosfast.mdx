## Поиск CAN ID

:::предупреждение
* Система FLYOS-FAST по умолчанию имеет скорость CAN 1М
* Убедитесь, что скорость сети CAN на верхнем уровне, мостовая плата CAN и скорость CAN на инструментальной плате полностью совпадают
:::

    1. В SSH введите команду `lsusb`, нажмите клавишу `Enter`. Если появится изображение `1d50:606f`, можно переходить к следующему шагу.
    <ImageView image={require('@site/docs/General/get-id/img/606f.webp').default} size="100%" align="center" />
    2. Где `1d50:606f` является устройством, которое вы будете использовать.
    3. Если подключено несколько мостовых плат с прошивкой USB Bridge CAN или нескольких UTOC и т. д., то могут появиться несколько устройств `1d50:606f`, необходимо исключить до одного устройства.
    4. Если нет `1d50:606f`, проверьте правильность режима переключателя на материнской плате, попробуйте повторно скомпилировать и записать прошивку USB Bridge CAN, попробуйте подключить 24В питание к материнской плате.

    :::предупреждение Внимание
    - Должен быть найден `1d50:606f` перед выполнением следующего действия
    - И должно быть только одно устройство `1d50:606f`
    :::

    5. Введите следующую команду для поиска ID:
    ```bash
    python ~/klipper/scripts/canbus_query.py can0
    ```
        * Если появляется UUID, после `Application:` показывает `Klipper`, то этот ID можно использовать напрямую
        * Если появляется UUID, после `Application:` показывает `CANBOOT` или `Katapult`, то для использования необходима прошивка USB Bridge CAN
        <ImageView image={require('@site/docs/General/get-id/img/canbus-uuid.webp').default} size="80%" align="center" />

    :::предупреждение Внимание
    - Только при появлении `UUID` и после `Application:` показывает `Klipper` можно переходить к следующему шагу
    :::

    :::опасность Важно
    * Если не удается найти CAN ID, см. :<Button variant="contained" disableElevation component="a" rel="noopener" href="/docs/DebugDoc/faq/klipper_can">Сборник проблем CAN</Button>
    :::

## Заполнение CAN ID

    1. Откройте веб-интерфейс верхнего уровня, найдите в боковом меню конфигурационные опции:

    <table>
      <tr>
        <td>
          - Для fluidd: `{…}`, нажмите, чтобы найти `printer.cfg`
          <ImageView image={require('@site/docs/General/get-id/img/fluidd-cfg.webp').default} size="100%" align="center" />
        </td>
        <td>
          - Для mainsail: `Машина`, нажмите, чтобы найти `printer.cfg`
          <ImageView image={require('@site/docs/General/get-id/img/mainsail-cfg.webp').default} size="100%" align="center" />
        </td>
      </tr>
    </table>

    2. Скопируйте найденный `UUID` и в файле `printer.cfg` найдите опцию `[mcu]`, затем внизу добавьте `canbus_uuid: ` с найденным `UUID`, сохраните и перезагрузите систему для подключения к материнской плате.

    :::опасность
    * Пример ниже показывает ситуацию с **одной материнской платой с прошивкой USB Bridge CAN** и **одной CAN инструментальной платой**, подключенными вместе!!
    * Настройки UUID приведены только в ознакомительных целях!! Пожалуйста, строго следуйте указанному формату при заполнении UUID
    * Формат UUID материнской платы CAN:
     ```
    [mcu]  
    canbus_uuid: ********
    ```

    * Формат UUID инструментальной платы CAN:  
    ```
    [mcu имя_инструментальной_платы]  
    canbus_uuid: ********
    ```    
    :::

        <ImageView image={require('@site/docs/General/get-id/img/canid.webp').default} size="50%" align="center" />


    3. Если Klipper выводит сообщение `ADC out of range`, это нормальное явление. Подключите горячий стол и термистор к материнской плате и настройте пины термистора для головки и горячего стола, затем сохраните и перезагрузите систему.

        :::опасность

        Важно: Все ID, указанные в документации, являются примерами. На самом деле каждый ID материнской платы уникален. Пожалуйста, измените и заполните согласно фактически полученному ID.

        :::
