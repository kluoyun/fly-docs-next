# ファームウェア焼付け説明
    
    * ファームウェアのコンパイルと焼付けには上位機器が必要です
    * SHT36 V3はRS232とCANの2つの接続方法で上位機器と接続できます
    * RS232とCANは`XT30(2+2)-F`を使用して接続する必要があります
    * 下記は`XT30(2+2)-F`データラインのスタイルです
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意
* メインボードのファームウェア焼付けにはKatapultを介してKlipperを焼付けます
* メインボードにKatapultファームウェアが既に焼付けられていることが確認できる場合、最後のKatapultファームウェア焼付けは必要ありません。それはメインボードのKatapultファームウェア焼付けが失敗した場合の予防措置です
* メインボードにリセットボタンを2回素早く押すと、メインボードにLEDが点滅しているか確認できます。もしそうなら、メインボードがKatapultファームウェアに入っていることを意味します
:::

## どの種類のファームウェアが必要かを判断する方法

    * SHT36 V3は以下の方法で上位機器と接続できます
    * <Button variant="contained" disableElevation href="./can">CANファームウェア</Button>
        * 好み：`Katapult`ファームウェアを焼付けた後、マザーボードに触れる必要なくファームウェアを更新でき、バス接続をサポートします
        * 悪い点：`KlipperScreen`と一緒に使用すると、ホーム位置のタイムアウトエラーなどが発生する可能性があります
        * ヒント：CANを使用する場合は、上位機器がCANに対応しているか確認し、UTOCまたはブリッジファームウェアを焼付けたマザーボードを合わせて使用してください
    * <Button variant="contained" disableElevation href="./rs232">RS232ファームウェア</Button>
        * 好み：特別なソフトウェアが不要で、通信が信頼性があり安定しています
        * 悪い点：ファームウェアを再焼付けするには手動DFUに入ることか、RSTをダブルクリックして`Katapult`に入らないと更新できません
        * ヒント: G2TまたはUTORを使用してのみ機能します
