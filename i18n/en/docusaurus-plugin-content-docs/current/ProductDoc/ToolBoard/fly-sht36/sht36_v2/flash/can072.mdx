# 072 Version CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select **GPIO pins to set at micro-controller startup**, input: !PC13, please note it should be in uppercase.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/072.png').default} size="100%" align="center" />
    * Press `Q`, then you will see **Save configuration**, press `Y` at this moment.
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * You should now have saved the configuration and exited to the command line interface.
    * Input `make -j4` to start compiling, it takes some time.
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * If the following output appears, the compilation is successful.

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Burn Firmware

    * Before burning the firmware, connect the tool board to UTOC or a mainboard with bridge firmware using an XT30 cable. Below is a reference wiring diagram.
    * Please ensure that the DIP switches are set correctly.
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Ensure that the resistance between CAN H and CAN L is around `60立` when measured with a multimeter in a power-off state. If it is `120立`, you need to connect a `120立` jumper wire. If it is around `40立`, remove the jumper wire.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.png').default} size="50%" align="center" />
