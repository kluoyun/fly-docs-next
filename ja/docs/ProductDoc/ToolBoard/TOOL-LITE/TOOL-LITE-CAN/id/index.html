  • CAN HとCAN Lの抵抗値が60Ω付近であることを確認してください。もしそれが120Ωなら120Ωジャンパーキャップを挿入し、40Ω付近ならジャンパーキャップを挿入しないでください

    CAN設定とID検索​

    • CANを使用する場合は、上位機がCAN設定を行っていることを確認してください。
    注意事項

    マザーボードが上位機に接続された状態で次の操作を実行してください。

    手記

    上位機のcanネットワーク、ブリッジボードのCANレート、CANツールボードのレートが完全に一致していることを確認してください。

    設備検索前の注意事項​

    • CAN IDを検索する前に、SSHに接続してください
    • ネットワーク経由でSSHにログインしていることを確認してください（シリアルポートでのログインではありません）
    • UTOCを使用しているか、CANブリッジファームウェアが刷られているマザーボードが接続され、上位機とのデータ転送が可能なデータラインを使用していることを確認してください

    設備の検索​

    • 正常な上位機にログインしたら、lsusbコマンドを入力してデバイスを検索します。以下のいずれかの状況が発生します。
      • lsusbコマンドを入力するとlsコマンドが見つからないと表示される場合、以下のコマンドを入力してインストールできます。 
        sudo apt-get install usbutils
      • lsusbコマンドを入力しても何も反応がない場合、これはシステムの問題です。別のシステムを使用するか、正常に動作するシステムを使用してください。
      • 下の画像のような情報を得た場合、これは参考情報です。1d50:606fが表示されていることを確認してください。
    • 1d50:606fは今回使用する予定のデバイスです。後のメッセージは無視してください。システムの問題により、すべてが正しく表示されない場合があります。
    • 複数の1d50:606fがある場合、片方を除外することをお勧めします。それ以外の場合、後続の焼込やファームウェア接続に影響を与える可能性があります。例えば、FLY MINI PADの場合、搭載されたUTOCを使用し、他のCANブリッジデバイスは使用しないでください。
    • もしこれがない場合、データラインが正常に接続されているか、ファームウェアが正しいかどうかを確認してください。
    注意事項

    1d50:606fが表示されたら、CAN IDの検索に進んでください。

    CAN IDの検索​

    • CAN IDを検索するには、以下のコマンドを入力します。
    ~/klippy-env/bin/python ~/klipper/scripts/canbus_query.py can0
    • IDが表示され、最後のApplication:がKlipperと表示された場合、このIDはそのまま使用できます。
    • IDが表示され、最後のApplication:がCANBOOTまたはKatapultと表示された場合、ファームウェアを焼く必要があります。

    CANに関する問題集​

    注意事項
    • CAN IDが見つからない場合、以下のドキュメントを参照してください
    • CAN問題集

    CAN IDを構成ファイルに記述する​

    • プリンタのWebインターフェースを開き、左側のサイドバーにある構成オプションで、fluiddでは{…}、mainsailではマシンを選択してクリックします。これにより、printer.cfgが表示されます。

    • 検索したUUIDをコピーし、printer.cfgの[mcu]セクションのcanbus_uuid:に貼り付けます。保存して再起動すると、マザーボードに接続されます。Klipperがファームウェアの更新が必要、ADCエラーなどの警告を出す場合、Klipperがマザーボードに接続されていることを示します。

    • Application:がCanbootの場合、構成ファイルに記述することはできません。Klipperファームウェアを焼くことで次のステップに進めます。

      危ない

      注意：ドキュメントで示されているすべてのIDは例です。実際の各マザーボードのIDは異なりますので、実際に取得したIDに基づいて修正してください。

    • 参考となる構成例

      危ない

      • マザーボードのIDは必ず[mcu]に加え、canbus_uuid:を追加して検索したIDを指定します。
      • ツールボードの場合は必ず[mcu ツールボード名]に加え、canbus_uuid:を追加して検索したIDを指定します。
