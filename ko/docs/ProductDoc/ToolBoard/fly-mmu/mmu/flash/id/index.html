번역 중에 설정을 변경하지 마십시오.

주의

번역 중에 설정을 변경하지 마십시오.

  • USB ID 가져오기
  • CANBus UUID 가져오기
  • RS232 ID 가져오기

  • 데이터 전송 기능이 있는 데이터 케이블을 준비하십시오. 길이가 매우 긴 경우 자석 링이 있는 데이터 케이블을 사용하는 것이 좋습니다.

    USB 검색 가이드​

    장치 검색 전 주의사항​

    • USB ID를 검색하기 전에 SSH 연결을 먼저 수행하세요.
    • 네트워크를 통해 SSH에 로그인하고 시리얼 포트를 통해 로그인하지 않는지 확인하세요.
    • USB 펌웨어가 프로그래밍된 보드가 제대로 연결되어 있고 상위 기기와 데이터 전송 기능이 있는 데이터 케이블로 연결되어 있는지 확인하세요.

    장치 검색​

    • 상위 기기가 정상적으로 인터넷에 연결되어 있고 하위 기기와 데이터 전송 기능이 있는 데이터 케이블로 연결되어 있는지 확인하세요.
    • SSH를 통해 상위 기기에 접속한 후 lsusb 명령어를 입력하고 Enter를 누르세요. 아래 이미지에서 강조된 정보가 표시되면 다음 단계로 진행할 수 있습니다.
      • 아무런 정보도 반환되지 않으면, 이는 상위 기기의 시스템 문제입니다. 다른 시스템이나 상위 기기를 사용해야 합니다.
      • lsusb 명령어가 없음을 알리는 메시지가 표시되면 다음 명령어를 실행하여 설치할 수 있습니다. 
        sudo apt-get install usbutils
    • 1d50:6177는 이번에 사용할 장치입니다. OpenMoko, Inc. stm32f407xx는 참고용이지만 판단 기준은 아닙니다.
      • 일부 상위 기기는 시스템 문제로 인해 모든 정보를 표시하지 못하거나 표시하지 않을 수 있습니다.
      • lsusb 명령어를 입력하면 장치가 표시되지만 1d50:6177가 표시되지 않으면 데이터 케이블을 교체하거나 보드를 상위 기기의 다른 USB 포트에 연결해보세요.
    • 위의 단계를 수행했고 Klipper 펌웨어를 성공적으로 프로그래밍했으며 보드가 정상 작동하는 경우만 펌웨어 업데이트를 진행하려면 이 페이지 오른쪽 메뉴에서 펌웨어 업데이트 섹션을 참조하세요.
    주의사항

    1d50:6177 장치를 찾은 후에만 다음 단계로 진행할 수 있습니다.

    USBID 검색​

    ls /dev/serial/by-id/*
    • 아래 명령어를 입력하고 Enter를 누르세요. 모든 것이 정상적으로 작동하면 파란색 ID 한 줄이 표시됩니다.(아래는 예시입니다.)
    • 아래 명령어를 입력했음에도 ID가 나오지 않고 lsusb에는 장치가 표시되는 경우에는 상위 기기를 교체하거나 상위 기기의 시스템을 교체하세요.
    주의사항

    여러 개의 ID가 표시되면 보드의 MCU 모델에 따라 적절한 ID를 선택하세요.

    USB ID를 구성 파일에 입력​

    • 프린터 웹 인터페이스를 열고 왼쪽 사이드바의 구성 옵션에서 fluidd는 {…}; mainsail은 머신을 찾아 클릭합니다. 그러면 printer.cfg를 찾을 수 있습니다.

    • 파란색 ID를 복사하여 printer.cfg 구성 파일의 [mcu] 섹션의 serial:에 입력합니다. 저장 및 재부팅 후 보드에 연결됩니다. Klipper가 ADC out of range를 알리는 경우 이는 정상적인 현상이며, 열판과 온도센서를 제대로 연결하고 노즐, 열판의 온도센서 핀과 출력 핀을 구성한 후 저장 및 재부팅하면 됩니다.

    위험

    주의: 문서에 나온 모든 ID는 예시입니다. 각 보드의 실제 ID는 모두 다르므로 실제로 얻은 ID에 따라 수정하여 입력해야 합니다.

  • ID를 검색하기 전에 XT30 데이터 케이블을 사용하여 도구 보드를 UTOC 또는 다리 펌웨어가 설치된 메인보드에 연결합니다. 아래는 참조 연결 그림입니다.
  • 다이얼 스위치가 해당 위치에 있는지 확인하십시오.
  • CAN H와 CAN L가 전원이 꺼진 상태에서 약 60Ω 정도의 저항 값을 가짐을 확인하십시오. 만약 120Ω라면 120Ω 점프를 연결해야 합니다. 만약 40Ω 정도라면 점프를 제거해야 합니다.

    CAN 설정 및 ID 검색​

    • CAN을 사용하려면 상위 컴퓨터에 CAN 설정이 되어 있는지 확인하세요.
    주의

    메인보드가 상위 컴퓨터와 정상적으로 연결되어 있는지 확인한 후 아래 작업을 진행하세요.

    경고

    상위 컴퓨터의 CAN 네트워크, 다리 보드의 CAN 속도, CAN 도구 보드의 속도가 완전히 일치하는지 확인하세요.

    장치 검색 전 주의사항​

    • CAN ID를 검색하기 전에 SSH에 연결하세요.
    • 네트워크를 통해 SSH에 로그인하고 있는지 확인하세요. 시리얼 포트를 통해 로그인하지 않았는지 확인하세요.
    • UTOC를 연결하거나 CAN 브리지 펌웨어를 플래시한 메인보드가 연결되어 있고, 상위 컴퓨터로 데이터를 전송할 수 있는 데이터 케이블이 연결되어 있는지 확인하세요.

    장치 검색​

    • 이제 정상적으로 상위 컴퓨터에 로그인했으므로 lsusb 명령어를 입력하여 장치를 검색할 수 있습니다. 다음과 같은 상황 중 하나가 나타날 것입니다.
      • lsusb 명령어를 입력하면 ls 명령어가 없다는 메시지를 받게 된다면 아래의 명령어를 입력하여 명령어를 설치할 수 있습니다. 
        sudo apt-get install usbutils
      • lsusb 명령어를 입력하면 아무런 반응이 없을 경우, 시스템 문제로 인해 해결할 수 없습니다. 다른 시스템으로 교체하거나 정상적인 시스템을 사용하세요.
      • 아래 이미지에서 제공된 정보가 나타납니다. 이는 참고용입니다. 1d50:606f가 나타나는지만 확인하세요.
    • 1d50:606f는 이번에 사용할 장치입니다. 뒤의 메시지는 시스템 문제로 인해 일부만 표시되거나 전혀 표시되지 않을 수 있으므로 무시하세요.
    • 여러 개의 1d50:606f가 있다면 하나를 제외하는 것이 좋습니다. 그렇지 않으면 나중에 프로그래밍 및 펌웨어 연결에 영향을 미칠 수 있습니다. 예를 들어 FLY MINI PAD는 보드 내장 UTOC를 사용하도록 권장하며, 다른 CAN 브리지 장비는 사용하지 마세요.
    • 없다면 데이터 케이블이 제대로 연결되었는지, 펌웨어가 올바르게 플래시되었는지 확인하세요.
    주의

    1d50:606f가 나타날 때까지 CAN ID를 검색하지 마세요.

    CAN ID 검색​

    • ID 검색을 위해 아래 명령어를 입력하세요.
    ~/klippy-env/bin/python ~/klipper/scripts/canbus_query.py can0
    • ID가 나타나고 마지막에 Application:이 Klipper라고 표시되면 이 ID는 바로 사용할 수 있습니다.
    • ID가 나타나고 마지막에 Application:이 CANBOOT 또는 Katapult라고 표시되면 펌웨어를 업데이트해야 사용할 수 있습니다.

    CAN ID를 구성 파일에 기입​

    • 프린터 웹 페이지 인터페이스를 열고, 좌측 메뉴의 구성 옵션에서 fluidd일 경우 ...를, mainsail일 경우 기계를 클릭하여 printer.cfg를 찾으세요.

    • 검색된 UUID를 복사하여 printer.cfg 구성 파일의 [mcu] 섹션에 canbus_uuid: 다음에 작성하세요. 저장하고 재부팅하면 메인보드와 연결됩니다. Klipper가 펌웨어 업데이트 필요, ADC 오류 등과 같은 메시지를 표시한다면 Klipper가 메인보드와 연결되었다는 것을 의미합니다.

      위험

      주의: 문서에 나온 모든 ID는 예시이며, 각 메인보드의 ID는 모두 다릅니다. 실제로 얻은 ID를 바탕으로 수정하여 작성하세요.

  • ID를 검색하기 전에 XT30 데이터 케이블을 사용하여 도구 보드를 UTOR 또는 G2T에 연결합니다. 아래는 참조 연결 그림입니다.
  • 다이얼 스위치가 해당 위치에 있는지 확인하십시오.

    RS232 ID 검색​

    검색 전 주의사항​

    • RS232 ID를 검색하기 전에 먼저 SSH 연결을 해주세요.
    • 네트워크를 통해 SSH에 로그인했는지 확인하세요. 시리얼 포트로 로그인하지 않았는지 확인하세요.
    • UTOR가 연결되었는지 확인하고, 상위기기와 연결된 데이터선이 데이터 전송 기능을 가지고 있는지 확인하세요.
    • G2T를 사용한다면 UART 연결을 지원하는 시스템인지 확인하세요.

    장치 검색​

    • 이제 정상적으로 상위기에 로그인한 경우 lsusb 명령어를 입력하여 장치를 검색할 수 있습니다. 아래 중 하나의 상황이 발생할 것입니다.
      • lsusb 명령어를 입력하면 ls 명령어를 찾을 수 없다는 메시지가 표시되는 경우, 다음 명령어를 입력하여 명령어를 설치할 수 있습니다. 
        sudo apt-get install usbutils
      • lsusb 명령어를 입력했지만 아무런 반응이 없는 경우, 이는 시스템 문제입니다. 이에 대해 할 수 있는 것이 없으므로 다른 시스템으로 변경하거나 제대로 작동하는 시스템을 사용해야 합니다.
      • 아래 이미지에 나와 있는 정보가 표시됩니다. 이는 참고용입니다. ID 뒤의 문자만 확인하면 됩니다.
    • 1a86:7523는 이번에 사용할 장치입니다. 뒤의 메시지는 시스템 문제로 인해 완전히 표시되지 않을 수도 있으므로 신경 쓸 필요가 없습니다.
    • 그렇지 않다면, 데이터선이 잘 연결되었는지, 펌웨어가 올바르게 업데이트되었는지 확인하세요.
    주의사항

    1a86:7523가 있을 때만 아래 명령어를 사용하여 RS232 ID를 검색할 수 있습니다.

    RS232 ID 검색​

    ls /dev/serial/by-path/*
    • ls /dev/serial/by-path/* 명령어를 입력했지만 ID가 나오지 않는 경우, lsusb에는 장치가 있을 때 상위기기를 변경하거나 상위기기 시스템을 변경하세요.
    • UTOR를 사용할 때 세 개의 ID가 출력될 수 있으며, 다른 UTOR 시스템을 사용한 적이 있다면 여러 개의 ID가 나타날 수 있습니다.
    • id 뒤에 -port0가 붙어 있는지 확인하세요. 그렇지 않으면 사용할 수 없습니다.
    주의사항

    도구판과 UTOR가 반드시 접지되어 있어야 합니다! 그렇지 않으면 도구판에 연결할 수 없습니다!

    RS232 ID를 구성 파일에 작성​

    • 프린터 웹 인터페이스를 열고, 왼쪽 사이드 바의 구성 옵션에서 fluidd는 {…}이고, mainsail은 기계를 클릭하여 printer.cfg를 찾을 수 있습니다.

    • 파란색 ID를 복사하여 printer.cfg 구성 파일의 [mcu SHT36] 섹션의 serial:에 작성합니다. 저장하고 재시작하면 주보드에 연결할 수 있습니다. Klipper가 ADC out of range라는 메시지를 표시하는 것은 정상적인 현상입니다. 열판과 열감지기를 잘 연결하고, 노즐, 열판의 열감지기 핀과 출력 핀을 구성한 후 다시 저장하고 재시작하면 됩니다.

    주의사항
    • RS232 ID는 baud:와 restart_method:command도 구성해야 합니다.
    • 아래는 참고용 구성입니다.
    [mcu SHT36]
    serial: /dev/serial/by-path/platform-1c1c000.usb-usb-0:1.2:1.0-port0
    baud: 250000
    restart_method:command
    위험

    주의: 문서에 나타나는 모든 ID는 예시이며, 각 주보드의 ID는 모두 다릅니다. 실제로 얻은 ID에 따라 수정하여 작성해야 합니다.

    위험

    아래 ID는 사용할 수 없습니다!!

    serial: /dev/serial/by-id/usb-1a86_USB_Serial-if00-port0
